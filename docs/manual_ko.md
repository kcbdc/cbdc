# CBDC 위기관리 시뮬레이터 사용 매뉴얼 (한국어)

버전: v2.0 · 논문 R21 기반  
대상: CBDC 정책 연구자, 중앙은행 실무자, 금융안정 분석가

---

## 1. 이 시뮬레이터는 무엇인가요?

이 도구는 **CBDC(중앙은행 디지털화폐) 설계 방식이 금융위기 시 은행 시스템 안정성에 어떤 영향을 주는지** 시뮬레이션합니다.

논문 "Central Bank Digital Currency Design, Systemic Risk, and Financial Stability" (Journal of Financial Stability 투고)의 분석 엔진을 웹에서 직접 실행할 수 있도록 만든 도구입니다.

**핵심 연구 결과 (논문 Table 4):**
- Dynamic Cap + 계층금리 설계가 실패확률을 **27.2%p 감소** (0.4118 → 0.1400)
- 보유한도 없는 CBDC가 가장 위험
- 설계 순위: DynCap+Tiered < Fixed Limit < Unconstrained CBDC

## 3. 파라미터 설명 (쉬운 설명)

### 📌 국가 선택
한국, 미국, 유럽연합, 중국, 스웨덴 중 선택합니다.  
각 국가마다 은행 네트워크 밀도가 다르게 설정됩니다.
- 한국: 밀도 0.82 (높음 → 전파 빠름)
- 스웨덴: 디지털 결제율 0.95 (가장 높음)

### 📌 충격 강도 (S)
**쉬운 설명:** 금융위기가 얼마나 심각한지를 나타냅니다.
- 0.0 = 위기 없음
- 0.35 = 보통 수준 (기본값)
- 0.75 = 심각한 위기 (논문 Table 4 기준)
- 1.0 = 최악의 위기

### 📌 디지털 결제율 (D)
**쉬운 설명:** 국민이 디지털로 결제하는 비율입니다. 높을수록 CBDC로 자금이 빠르게 이동할 수 있습니다.
- 한국은 0.92 (92%가 디지털 결제 사용)

### 📌 은행 집중도 (C)
**쉬운 설명:** 소수 대형 은행이 시장을 얼마나 장악하는지입니다.  
높으면 한 은행이 무너질 때 전체로 퍼지기 쉽습니다.

### 📌 현금 완충비율
**쉬운 설명:** 사람들이 보유하는 현금 비율입니다.  
높으면 CBDC 이탈을 어느 정도 막아줍니다.

### 📌 보유한도 (KRW)
**쉬운 설명:** 한 사람이 CBDC를 얼마까지 보유할 수 있는지 상한선입니다.  
낮을수록 뱅크런 위험이 줄어들지만, 편의성도 줄어듭니다.
- 논문 최적값: **5.5M KRW** (약 4,000달러)
- 기본값: 20M KRW

### 📌 Dynamic Cap (동적 한도)
**쉬운 설명:** 위기 상황에서 자동으로 보유한도를 낮추는 기능입니다.  
활성화하면 뱅크런 시 CBDC 유출을 자동으로 제한합니다.

### 📌 LOLR (최종대부자)
**쉬운 설명:** 중앙은행이 위기 시 긴급 자금을 공급하는 기능입니다.  
스트레스가 0.45를 넘으면 자동으로 개입합니다 (논문 Section 3.1).

### 📌 Tiered Rate (계층금리)
**쉬운 설명:** CBDC를 많이 가질수록 낮은 이자를 주어 과도한 보유를 억제합니다.

### 📌 Monte Carlo 횟수
**쉬운 설명:** 같은 시나리오를 몇 번 반복 실행할지입니다.  
많을수록 정확하지만 느립니다. 기본값 100회 권장.

## 4. 결과 읽는 법

### 실패확률 P(Fail)
은행 시스템이 붕괴할 확률입니다.
- 0.0 = 0% (완전 안전)
- 0.14 = 14% (논문: 한국 DynCap+Tiered 최적)
- 0.41 = 41% (논문: 무제한 CBDC)

### 안정성 (Stability)
수식: `1 / (1 + exp(13 × (stress - threshold)))` (논문 Eq.1)  
높을수록 좋습니다. 0.5를 넘으면 안정 상태입니다.

### 스트레스 (Stress)
금융 시스템 전체 압박 지수입니다. 낮을수록 좋습니다.

### 후생지수 (Welfare)
편의성(결제 효율, 금융포용, 혁신) – 위험(실패, 분기, 운영 리스크)을 합산한 지수입니다.  
수식 (논문 Section 3.6):
```
W = 0.25·PayEff + 0.10·Inclusion + 0.15·Innovation 
    - 0.30·Risk - 0.15·Disintermediation - 0.05·Oper + 0.30
```

### 스트레스 막대차트
90일간 스트레스 추이를 보여줍니다. 막대가 높을수록 그날 위험이 큽니다.

### 주요 기여요인
어떤 요인이 실패확률을 높이거나 낮추는지 보여줍니다.
- 🔴 빨간 테두리 = 위험 증가 요인
- 🟢 초록 테두리 = 위험 감소 요인

## 8. 자주 묻는 질문

**Q: 시뮬레이터 결과가 논문과 다른 이유는?**  
A: 논문은 Python 기반 MT19937 난수 엔진(seed=20260618)을 사용하고, N=300 Monte Carlo 평균을 사용합니다. 웹 시뮬레이터는 브라우저에서 실행되는 단순화 버전입니다. 설계 순위(어떤 설계가 더 안전한지)는 일치합니다.

**Q: 보유한도를 5.5M KRW로 설정하면?**  
A: 논문 Table 7a에서 한국 기준 위험 최소화 임계값입니다. 다만 한국 예금보험 한도(5,000만 KRW)의 1/9 수준이라 실제 정책 적용 시 종합적 검토가 필요합니다.

**Q: 스웨덴에서 어떤 설계도 안전하지 않은 이유?**  
A: 스웨덴은 디지털 결제율(D=0.95)이 가장 높고 임계값(θ=0.60)이 낮아, 모든 캡 수준에서 위험이 높게 나타납니다. 논문 Section 6.6 참조.
